# Specification: Multi-language Support (i18n)

## Goal
Provide a seamless way for users to switch between Spanish and English, with automatic detection based on browser settings.

## Technical Strategy
- **Framework:** Next.js (App Router).
- **State Management:** React Context (`LanguageProvider`).
- **Persistence:** `localStorage` for user preference.
- **Detection:** `navigator.language` on first visit.
- **Translation Files:** JSON files in `app/_locales/`.

## Components
### LanguageProvider
- Wraps the application.
- Exports `useLanguage` hook.
- Provides `t(key)` function for translations.

### LanguageSwitcher
- UI component to toggle between ES and EN.
- Placed in the Sidebar.

## Data Structure
`app/_locales/es.json` & `app/_locales/en.json`:
```json
{
  "nav": {
    "home": "Inicio",
    "blog": "Blog",
    "projects": "Proyectos"
  },
  ...
}
```

## Implementation Details
1. Create `LanguageProvider.tsx`.
2. Create `es.json` and `en.json`.
3. Update `layout.tsx` to use the provider and set `<html lang="...">`.
4. Replace hardcoded strings with `t()` calls.
